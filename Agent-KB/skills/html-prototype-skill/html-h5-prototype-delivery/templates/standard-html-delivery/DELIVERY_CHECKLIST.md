# Delivery Checklist

## Runtime

- [ ] `index.html` opens directly in a browser.
- [ ] `npm run dev` starts a local preview server.
- [ ] All CSS and JS files load with relative paths.
- [ ] No console `SyntaxError` or `TypeError`.

## UI

- [ ] Main page renders without blank screen.
- [ ] Navigation works.
- [ ] Buttons, tabs, modals, toast, and sheets are clickable.
- [ ] No horizontal overflow on mobile canvas.
- [ ] Text does not overflow or overlap.
- [ ] Empty, loading, disabled, and error states are represented where needed.

## Annotation And Review

- [ ] Right annotation editor is visible.
- [ ] Current page and current scenario are shown.
- [ ] Notes explain business meaning, interaction rules, state rules, and acceptance criteria.
- [ ] State controls affect the prototype only when intended.
- [ ] PM notes and reviewer notes are separated if both are used.

## Documentation

- [ ] `docs/requirement.md` is filled.
- [ ] `docs/interaction.md` is filled.
- [ ] `docs/state-matrix.md` is filled.
- [ ] `docs/permissions.md` is filled.
- [ ] `docs/api-fields.md` is filled.
- [ ] `docs/tracking.md` is filled.
- [ ] `docs/copywriting.md` is filled.
- [ ] `docs/edge-cases.md` is filled.
- [ ] `docs/changelog.md` includes the latest change.

## Assets

- [ ] Required images are in `assets/images/`.
- [ ] Required icons are in `assets/icons/`.
- [ ] Required videos are in `assets/videos/`.
- [ ] No deliverable asset uses a local absolute path.

## Handoff

- [ ] Key screenshots are saved in `screenshots/`.
- [ ] The prototype scope and non-production nature are clear.
- [ ] Known risks or incomplete areas are documented.
