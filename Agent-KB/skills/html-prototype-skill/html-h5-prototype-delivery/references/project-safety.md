# 路径与版本边界

## 开始前

每次开始前先确认：

- 当前 `pwd`
- 用户指定的项目根目录
- 当前主版本目录
- 浏览器打开的 URL 是否指向主版本

如果当前路径和用户指定路径不一致，停止并说明，不要修改文件。

## 主版本优先

只修改用户明确指定的主版本目录。

凡是名字包含以下含义的目录，都视为备份或历史版本，除非用户明确要求，否则不要读取、运行或修改：

- `backup`
- `broken`
- `checkpoint`
- `bad`
- `restore`
- `old`
- 日期时间戳版本
- 非精确主目录名的相似目录

## 禁止误触碰

不要因为 Browser 当前打开了旧路径，就去修改旧目录。

预览、测试、导出都必须基于当前主版本目录。

## 输出要求

如果任务强调路径安全，完成后说明：

- 当前 `pwd`
- 实际修改的完整路径
- 是否所有修改都发生在主版本目录内
- 是否误触碰备份目录
