# 社交产品模式

仅在社交、泛娱乐、Dating、IM、陌生人社交、语音 / 视频互动、会员 / 金币产品中使用本 reference。

## 常见页面

- Discover / Feed / 推荐流
- Likes / 关系列表
- Chat / IM
- Profile / 用户资料
- Me / 账户中心
- Wallet / 余额
- Subscription / Membership

## 常见商业模式

- 会员订阅
- 金币 / 点数充值
- 礼物
- 单次解锁
- 私密内容
- 消息次数限制
- 曝光 / Boost
- 高级筛选
- 资料增强

不要把所有商业模式混成订阅。

## 聊天模式

IM 原型应明确：

- 普通消息
- 消息次数限制
- 对方主动发送内容
- 我方发送内容
- 私密照片 / 视频
- 倒计时
- 销毁
- 礼物
- 付费提示

不同链路要分开：

- 对方发给我，我可能需要付费解锁。
- 我发给对方，对方可能免费查看并倒计时销毁。

## 关系与权限

用户关系和权限要分清：

- 我喜欢谁
- 谁喜欢我
- 匹配
- 可聊天
- 可看高清
- 可解锁内容

`paid_user`、`subscriber`、`coin_user`、`low_balance_user` 是不同状态，不要混用。

## 原型注释器建议

社交产品右侧注释器优先展示：

- 用户类型
- 权限状态
- 次数状态
- 消耗状态
- 功能场景
- 模拟动作
- 备注

不要默认展示大量技术字段。
