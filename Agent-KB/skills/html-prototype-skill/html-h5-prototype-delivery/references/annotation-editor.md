# 原型注释器

## 定位

Annotation Editor 是产品原型说明引擎，不是调试面板。

它应该帮助 PM / 设计 / 开发理解：

- 当前页面是什么
- 当前功能场景是什么
- 用户类型是什么
- 状态变化如何影响原型
- 备注和验收是什么
- 哪些事件需要埋点

## 结构

右侧注释器建议包含：

- 当前页面
- 当前场景
- 用户类型
- 功能场景导航
- 状态 / 权限 / 消耗卡片
- 说明 / 状态 / 埋点 Tab
- PM 备注
- 查看者备注
- 导出操作

## 信息架构

不要按 UI 区块组织主信息，例如：

- Header
- Card
- Button group

应按业务功能场景组织，例如：

- 滑卡页面
- 二级页面
- 消息次数限制
- 私密内容发送
- 私密内容解锁
- 账户状态

## 技术字段

不要默认大面积展示：

- `currentPage`
- `coinBalance`
- `subscriptionStatus`
- `likesMePermissionStatus`
- `activeChatThreadId`
- 付费触发公式

技术字段可以保留在内部数据或折叠区。

## 原型联动

左侧原型进入二级页、对话框、设置页等真实页面时，右侧注释器应同步定位到对应页面 / 场景。

同步不能破坏左侧点击。

## 不允许

- 不要把注释器放回 phone frame 内部。
- 不要让注释器遮挡左侧原型。
- 不要为了精简注释器删除左侧真实路由。
- 不要让高亮层拦截点击。
