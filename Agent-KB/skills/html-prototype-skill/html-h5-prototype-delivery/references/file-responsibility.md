# 文件职责

## 推荐职责

- `index.html`：入口壳，只放根容器、CSS 链接、JS 引入、必要 meta。
- `style.css`：样式、布局、主题变量、组件样式、响应式、状态视觉。
- `state.js`：运行时状态、默认值、状态更新、派生状态、持久化 helper。
- `data.js`：mock 数据、资源路径、配置、文案、套餐、功能场景、注释配置。
- `app.js`：渲染、路由、事件绑定、交互、弹窗、Bottom Sheet、注释器渲染。
- `assets/`：图片、视频、图标等资源。

## 禁止

- 不要把拆分文件合并回一个 HTML。
- 不要在 `index.html` 写具体页面 UI。
- 不要在 `index.html` 写内联 CSS / JS。
- 不要把状态写进 `style.css`。
- 不要把大量静态 mock 数据硬编码到 render 函数里。
- 不要为了快速修复跨文件职责。

## 判断原则

如果一段代码描述“是什么数据”，放 `data.js`。

如果一段代码描述“现在是什么状态”，放 `state.js`。

如果一段代码描述“怎么显示 / 点击后做什么”，放 `app.js`。

如果一段代码描述“长什么样”，放 `style.css`。
