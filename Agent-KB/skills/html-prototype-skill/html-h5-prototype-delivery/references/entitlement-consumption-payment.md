# 权限 / 次数 / 消耗 / 支付

## 抽象模型

不要默认所有商业模式都是订阅。

常见类型：

- `entitlement`：会员、VIP、角色、订阅、权限。
- `counter`：次数、额度、每日限制、试用次数。
- `consumption`：金币、点数、单次解锁、礼物、私密内容。
- `cashier`：收银台、充值、购买套餐、支付结果。

## 规则

- 权限不足走权限 / 会员 paywall。
- 金币或点数不足走 recharge / cashier。
- 次数不足根据产品规则走订阅、次数包、充值或拦截，不要默认。
- `paid_user` 不等于 `subscriber`。
- `subscriber` 不一定免费拥有所有消耗型内容。
- 单次付费内容应独立于订阅权益建模。

## 状态联动

状态型控件更新后，左侧必须同步。

示例：

- 免费用户看到模糊。
- 订阅用户看到高清。
- 金币不足打开充值。
- 消耗成功扣除余额。
- 次数到 0 后拦截继续操作。

## 禁止

- 不要把所有付费入口都接到 subscription paywall。
- 不要把所有付费入口都接到 coin recharge。
- 不要写死用户权限。
