# User Profile 个人主页规范

> 状态：正式版基线生效中  
> 平台基线：Laka iOS 1.4.7  
> 主验收画布：440 x 956

## 1. 页面职责

个人主页把身份信息和社交证明转化为关系操作。页面首先展示“这个人”，随后清晰表达在线状态、媒体、兴趣、礼物、关系状态和当前允许操作。

## 3. 固定视觉合同

- 首屏以人物媒体为主：全宽照片或头像艺术图。
- 返回和更多/分享操作必须在媒体上清晰可见。
- 姓名、在线状态、ID、性别年龄、等级在 Hero 下部叠加，并保证对比度。
- 白色圆角内容面板与 Hero 发生叠压。
- `Profile`、`Relation`、`Moment` 是主内容切换。
- 他人主页的主操作固定在底部安全区上方。

## 4. 页面结构

1. Hero 媒体、返回、更多/分享。
2. 头像及可选媒体缩略图。
3. 身份叠层：姓名、国旗、认证、在线、ID、等级徽章。
4. 白色内容面板与可选社交计数。
5. Profile / Relation / Moment Tab。
6. Bio、可选语音简介、礼物墙、标签、基础信息。
7. 他人主页底部操作栏，或自己主页编辑操作。

## 5. 布局规则

- Hero 媒体必须可辨识，不得被超大卡片遮住。
- 身份文字背后必须有足够的明暗保护层。
- 缩略图带水平约束，不得把身份信息挤出屏幕。
- 内容面板顶部圆角工作范围 20-28 px。
- Tab 使用 Laka 紫色指示器，并在各状态下位置稳定。
- 礼物墙和标签允许横向滚动，但标题与数量始终可见。
- 底部操作位于手势安全区上方，且不遮挡正文。

## 6. 状态矩阵

| 状态 | 主操作要求 |
| --- | --- |
| 自己主页 | 编辑/分享，不显示关注聊天操作坞 |
| 无关系 | 按业务规则显示关注及建联/聊天入口 |
| 我已关注，等待对方 | 显示等待/已关注，禁止重复发起 |
| 对方已关注我 | 提供回关及符合条件的会话操作 |
| 已互关 | 显示互关关系并开放允许的社交操作 |
| 已同意聊天 | 显示直接会话操作 |
| 可视频 | 增加正式视频标识，不遮挡身份信息 |
| 已拉黑/受限 | 隐藏被禁止的操作，仅在允许时提供恢复入口 |

性别只改变内容，不改变个人主页组件系统。

## 7. 必须覆盖的状态

- 自己/他人、男性/女性、在线/最近在线/离线。
- 单图、多图、无图。
- Bio、礼物墙、标签为空或有内容。
- 有/无语音简介。
- 状态矩阵中的全部关注关系。
- 已同意聊天、请求中、已拉黑、关系已删除。
- 长阿拉伯语 Bio 和混合语言身份信息。
- 加载中、媒体失败与重试。

## 8. 交互规则

- 返回后保留来源列表位置。
- 点击缩略图切换 Hero 媒体，不改变身份叠层位置。
- Profile / Relation / Moment 只切换内容，不切换页面外壳。
- 关注、聊天、视频和关系操作必须立即反映当前权限与关系状态。
- 举报、拉黑等次级命令放入更多菜单。

## 9. 素材要求

- 真实用户媒体是运行时内容，不保存为设计素材。
- 头像框、徽章、关系标识、礼物和可视频标识必须来自正式来源。
- 原型需要头像时，只能使用已登记的成年 MENA 临时头像，不得使用截图人物。

## 10. 禁止继承

- 截图中的真实人物照片。
- 亮色媒体上低对比度白字。
- 固定高度 Bio 导致阿拉伯语被裁切。
- 底部操作栏遮挡页面内容。
- 与当前关系权限矛盾的标签或按钮。

## 11. 验收

- 他人主页与 M09、自身主页与 M07 并排对比。
- 验证状态矩阵中的每个关系状态。
- 验证长阿拉伯语、无礼物、有礼物、多媒体。
- 功能补丁前后，未涉及区域的 Hero、Tab 和底部操作几何必须一致。
- `baseline.html` 仅在当前需求要求原型时修改。
