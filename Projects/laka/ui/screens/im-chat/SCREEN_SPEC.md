# IM Chat 私聊页面规范

> 状态：正式版基线生效中  
> 平台基线：Laka iOS 1.4.7  
> 主验收画布：440 x 956

## 1. 页面职责

IM Chat 是一对一关系的核心承载页面。氛围要亲密、轻盈，同时必须清楚表达建联权限、付费能力、亲密度进度和消息状态。

- 本地基线：`reference.png`
- 默认聊天：M10，`im-chat-20260713-112615.png`
- 礼物面板：M11
- 表情面板：M12

  - `主态_对方点了聊天_同意了_点视频toast`

## 3. 固定视觉合同

- 使用浅粉紫关系背景，不使用宫殿主框架。
- 头部居中展示对方头像和身份，返回与更多分置两侧。
- 日期分隔弱化并居中。
- 收到消息按对方头像一侧对齐，发出消息镜像对齐。
- 亲密度入口可见但不得遮挡消息。
- 输入框和快捷回复位于键盘及安全区上方。

## 4. 页面结构

1. 安全区和对方头部。
2. 需要时显示建联/请求状态区。
3. 可滚动消息流。
4. 亲密度数值/入口。
5. 可选快捷回复。
6. 输入框和发送操作。
7. 工具行：受限媒体/语音、礼物、表情、关系工具。
8. 礼物、表情、媒体等可展开底部面板。

## 5. 布局规则

- 消息气泡随内容变化，但设定可读最大宽度。
- 图片、语音消息必须保持稳定比例和元信息位置。
- 连续收到消息时，对方头像对齐规则保持一致。
- 输入区是固定交互区域，不是覆盖消息的悬浮卡片。
- 键盘打开后，最新消息和输入区仍可见。
- 快捷回复可横向滚动或换行，不能压缩发送触控区。
- 阿拉伯语按内容方向排版，但发送者左右归属不变。

## 6. 建联状态矩阵

| 状态 | 必须行为 |
| --- | --- |
| 我主动发起聊天 | 显示请求/等待逻辑及当前可执行操作 |
| 对方请求聊天 | 明确显示同意、删除、拉黑选择 |
| 已删除 | 显示删除结果及正确恢复规则 |
| 已同意 | 开放符合权限的输入与媒体工具 |
| 已拉黑 | 禁止发送，并明确展示拉黑状态 |
| 同意后点击视频 | 展示正式权限或可用性 Toast |

未满足权限或亲密度阈值前，不得提前开放付费或私密工具。

## 7. 必须覆盖的消息状态

- 文本、图片、语音、礼物、系统消息。
- 发送中、已发送、失败、重试及支持时的删除消息。
- 新日期分隔、长历史、空白新会话。
- 请求中、已同意、已删除、已拉黑。
- 媒体/语音工具锁定和解锁。
- 礼物、表情、亲密度面板开关。
- 键盘开关、断网与重连。
- 长阿拉伯语和混合语言消息。

## 8. 交互规则

- 只有内容有效且权限允许时才启用发送。
- 点击锁定工具必须打开原因说明，不得成为无反馈操作。
- 礼物面板必须展示余额、价格、数量和最终发送操作。
- 表情按现有产品行为插入或发送。
- 打开亲密度规则不得丢失聊天状态。
- 举报、拉黑、删除等次级命令进入更多菜单。

## 9. 素材要求

货币、礼物、锁、亲密度和关系图标必须使用正式素材。M11 礼物图只作为证据，取得母版前禁止复用；截图中的对方照片禁止提取。

## 10. 禁止继承

- 截图中的真实用户媒体。
- 图片消息被裁切或控件覆盖图片。
- 锁定操作呈现为可用状态。
- 快捷回复覆盖输入框。
- 错误的亲密度文案或互相矛盾的阈值。

## 11. 验收

- 在 440 x 956 下与 M10-M13 并排对比。
- 验证六种建联状态和全部消息发送状态。
- 验证键盘、礼物、表情和亲密度面板切换。
- 打开/关闭面板后聊天滚动位置必须保留。
- `baseline.html` 仅在当前需求要求原型时修改。
