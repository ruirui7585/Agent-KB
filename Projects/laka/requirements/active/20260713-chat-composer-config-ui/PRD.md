# 女性用户通话权益配置 PRD

## 1. 需求范围

本需求定义女性用户被配置为 `核心女` 或 `屏蔽女` 后，前端在 IM 聊天页和个人主页中展示通话入口、视频权限标识和未解锁状态的规则。

该配置是女性用户维度的全局配置。无论用户处于主态还是客态，无论入口来自 IM 聊天页还是个人主页，都必须使用同一套配置结果。

## 2. 核心定义

| 名称 | 说明 |
| --- | --- |
| 核心女 | 具备完整语音和视频通话权益的女性用户。核心女默认拥有视频权限。 |
| 屏蔽女 | 仅开放语音能力、不开放视频能力的女性用户。 |
| 未解锁 | 当前查看方尚未满足通话解锁条件。 |
| 已解锁 | 当前查看方已经满足通话解锁条件。 |
| 视频权限认证徽标 | 个人主页资料区展示的视频权限标识，仅核心女展示。 |

## 3. 全局业务规则

| 规则 ID | 规则 |
| --- | --- |
| R1 | 核心女 / 屏蔽女是女性用户的全局配置，不因主态、客态、页面入口或查看方身份改变。 |
| R2 | 核心女默认拥有视频权限，不需要额外前端状态判断。 |
| R3 | 屏蔽女不展示视频入口，不展示视频权限认证徽标。 |
| R4 | 正常情况下同一女性用户不会同时命中核心女和屏蔽女。 |
| R5 | 如果服务端异常同时返回核心女和屏蔽女，前端按核心女权益展示。 |
| R6 | 通话是否可点击由解锁状态决定，用户类型只决定可展示的通话类型。 |
| R7 | 页面中的示例价格仅用于展示，不代表正式价格、扣费、收益或结算规则。 |

## 4. 服务端要求

### 4.1 下发范围

服务端需要在 IM 聊天页和个人主页相关接口中，下发同一女性用户的权益配置结果。

至少需要支持以下语义：

| 字段语义 | 类型 | 说明 |
| --- | --- | --- |
| 女性用户类型 | enum | `core` 表示核心女，`blocked` 表示屏蔽女，普通女性可为空或普通值。 |
| 通话解锁状态 | enum / boolean | 表示当前查看方与该女性用户之间是否已解锁通话能力。 |
| 锁定原因 | enum / string | 未解锁时可选下发，用于前端展示锁定提示。 |

字段命名由研发确定，但前后端语义必须保持一致。

### 4.2 服务端判定规则

| 场景 | 服务端应返回 |
| --- | --- |
| 女性用户是核心女 | 返回核心女标识。 |
| 女性用户是屏蔽女 | 返回屏蔽女标识。 |
| 女性用户同时命中核心女和屏蔽女 | 按核心女返回，或明确提供优先级结果给前端。 |
| 当前查看方未满足通话条件 | 返回未解锁。 |
| 当前查看方已满足通话条件 | 返回已解锁。 |

## 5. 前端展示规则

### 5.1 IM 聊天页底部工具栏

| 女性用户类型 | 解锁状态 | 底部工具栏展示 | 点击通话入口 |
| --- | --- | --- | --- |
| 核心女 | 未解锁 | 展示 5 个图标：Album、Voice、Gift、Audio Call、Video Call；Audio / Video 为锁定态。 | 不打开呼叫选择器，提示需要解锁。 |
| 核心女 | 已解锁 | 展示 5 个图标：Album、Voice、Gift、Audio Call、Video Call；全部为可点击态。 | Audio / Video 均可打开呼叫选择器。 |
| 屏蔽女 | 未解锁 | 展示 4 个图标：Album、Voice、Gift、Audio Call；隐藏 Video Call；Audio 为锁定态。 | 不打开呼叫选择器，提示需要解锁。 |
| 屏蔽女 | 已解锁 | 展示 4 个图标：Album、Voice、Gift、Audio Call；隐藏 Video Call；全部为可点击态。 | Audio 可打开呼叫选择器。 |

### 5.2 个人主页通话入口

| 女性用户类型 | 解锁状态 | 通话入口展示 | 点击行为 |
| --- | --- | --- | --- |
| 核心女 | 未解锁 | 保留当前锁定通话入口样式。 | 打开未解锁提示。 |
| 核心女 | 已解锁 | 展示视频 + 语音组合通话入口。 | 打开视频和语音相关呼叫选项。 |
| 屏蔽女 | 未解锁 | 展示置灰 Audio Call 入口。 | 打开未解锁提示。 |
| 屏蔽女 | 已解锁 | 展示高亮 Audio Call 入口。 | 打开语音相关呼叫选项。 |

### 5.3 个人主页视频权限认证徽标

| 女性用户类型 | 是否展示 | 点击行为 |
| --- | --- | --- |
| 核心女 | 展示 | 显示 `Officially verified`。 |
| 屏蔽女 | 不展示 | 无。 |

展示规则：

- 徽标属于女性用户全局权益标识，主态和客态保持一致。
- 徽标只表达该女性用户具备视频权限，不代表当前查看方已经解锁视频通话。
- 点击徽标不得触发通话选择器。
- 点击徽标不得改变解锁状态。

## 6. 呼叫选择器规则

### 6.1 IM 聊天页

| 触发入口 | 展示内容 |
| --- | --- |
| Audio Call | 展示 Audio Call 相关选项。 |
| Video Call | 展示 Video Call 相关选项。 |

### 6.2 个人主页

| 女性用户类型 | 解锁状态 | 展示内容 |
| --- | --- | --- |
| 核心女 | 已解锁 | 展示 Invite to Video Call、Video Call、Invite to Audio Call、Audio Call。 |
| 屏蔽女 | 已解锁 | 展示 Invite to Audio Call、Audio Call。 |
| 任意类型 | 未解锁 | 不展示呼叫选择器，展示未解锁提示。 |

价格说明：

- 页面中出现的价格、币种、Earn / Cost 文案仅作为展示占位。
- 正式价格、扣费、收益和结算规则不在本需求中定义。

## 7. 未解锁提示规则

| 页面 | 未解锁点击行为 |
| --- | --- |
| IM 聊天页 | 展示轻提示，不打开呼叫选择器。 |
| 个人主页 | 展示未解锁弹窗。 |

个人主页未解锁弹窗内容：

| 元素 | 文案 |
| --- | --- |
| 标题 | `Insufficient intimacy` |
| 说明 | `Unlock call function when your intimacy reaches 1, chatting and sending gifts can increase intimacy` |
| 按钮 1 | `Send a gift` |
| 按钮 2 | `Chat` |

## 8. 异常与兜底

| 异常场景 | 前端处理 |
| --- | --- |
| 同时返回核心女和屏蔽女 | 按核心女展示。 |
| 未返回用户类型 | 按普通女性用户现有逻辑展示。 |
| 未返回解锁状态 | 按未解锁处理。 |
| 已解锁但缺少可用通话类型 | 不展示不可用入口，避免点击无响应。 |
| 屏蔽女返回视频权限 | 以屏蔽女规则为准，不展示视频入口和视频权限徽标。 |

## 9. 验收标准

| ID | 验收项 |
| --- | --- |
| A1 | 核心女未解锁时，IM 聊天页展示 Audio Call 和 Video Call 锁定态。 |
| A2 | 核心女已解锁时，IM 聊天页展示可点击的 Audio Call 和 Video Call。 |
| A3 | 屏蔽女未解锁时，IM 聊天页隐藏 Video Call，仅展示锁定态 Audio Call。 |
| A4 | 屏蔽女已解锁时，IM 聊天页隐藏 Video Call，仅展示可点击 Audio Call。 |
| A5 | 核心女个人主页展示视频权限认证徽标，屏蔽女不展示。 |
| A6 | 视频权限认证徽标点击后只显示 `Officially verified`，不触发通话选择器。 |
| A7 | 核心女个人主页已解锁时，可进入视频和语音相关呼叫选项。 |
| A8 | 屏蔽女个人主页已解锁时，仅可进入语音相关呼叫选项。 |
| A9 | 任意女性用户未解锁时，点击个人主页通话入口展示未解锁弹窗。 |
| A10 | 同一女性用户在主态、客态、IM 聊天页和个人主页中的配置结果一致。 |
| A11 | 异常同时命中核心女和屏蔽女时，前端按核心女权益展示。 |
| A12 | 展示价格不进入正式价格、扣费、收益或结算逻辑。 |
