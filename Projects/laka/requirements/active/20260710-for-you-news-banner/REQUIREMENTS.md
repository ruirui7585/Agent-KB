> 本文档基于原型 `newsbanner.html`（prototypeId: `homepage-news-wall-v1`）反推梳理，作为该功能的完整需求规格，供产品、设计、开发交付使用。

## 1. 需求概述

### 1.1 一句话描述
在 Laka 社交 App 首页「For You」信息流顶部新增一个 **Laka News 关系新闻墙 Banner**，轮播展示用户间的关系流转事件（CP 结成、Guardian 守护等），点击进入「Laka News 事件墙」，再由事件墙下钻到对应的「关系排行榜」，形成 `首页 Banner → 事件墙 → 排行榜` 的三级关系内容消费闭环。

### 1.2 核心价值
- **激活关系玩法曝光**：把原本分散在不同入口的 CP / Guardian 关系事件聚合到首页黄金位，提升关系玩法的认知与参与。
- **延长内容消费深度**：通过 Banner → 事件墙 → 排行榜的三级跳转，让用户从「看到一条关系」到「围观整条关系链榜单」。
- **强化社交竞争与荣誉感**：排行榜 + 倒计时 + 领奖台，制造关系经营的竞争氛围。

---

## 2. 背景与目标

### 2.1 背景
Laka 是面向 MENA 市场的社交 App，已具备 CP（情侣/搭子）、Guardian（守护）等关系玩法。当前关系事件散落在各玩法入口内，曝光不足，新用户难以感知「关系」是产品的核心特色之一。

### 2.2 目标
| 维度 | 目标 |
|---|---|
| 曝光 | 首页 Banner 100% 覆盖 For You 页用户，关系事件日均曝光提升 |
| 点击 | Banner 点击率（进入事件墙）达到预期阈值 |
| 深度 | 事件墙 → 排行榜的下钻转化率达到预期 |
| 参与 | 排行榜「Build」按钮点击，带动关系建立转化 |

> 具体数值指标由运营/数据同学在需求评审时补充确认。

## 3. 名词解释

| 名词 | 含义 |
|---|---|
| **Laka News** | 本功能总称，即「关系新闻墙」，聚合展示用户间的关系流转事件 |
| **Banner** | 首页 For You 信息流顶部的关系事件轮播卡片 |
| **关系事件（Relation Event）** | 两个用户之间发生的一次关系建立/流转，如 Lina 与 Sara 结成 CP |
| **CP** | 情侣/搭子关系，图标 💞 |
| **Guardian** | 守护关系，图标 🛡️ |
| **事件墙（News Wall）** | Laka News 二级页，按关系类型分类展示事件卡片 |
| **排行榜（Ranking）** | 三级页，按 Daily/Weekly/Monthly 展示关系组合的积分排名 |
| **关系积分** | 衡量关系亲密度的分数，排行榜中以 🔥 数值展示（单位 m，即「万」） |

## 4. 用户角色与场景

### 4.1 目标用户
- MENA 市场 Laka 注册用户，有浏览内容、建立社交关系的诉求。
- 既包括「围观别人关系」的吃瓜用户，也包括「经营自己关系」的参与用户。

### 4.2 核心场景
| 场景 | 用户行为 | 期望结果 |
|---|---|---|
| 逛首页 | 滑动 For You 信息流 | 顶部看到关系事件轮播，感知到「关系」是产品特色 |
| 围观关系 | 点击 Banner | 进入事件墙，看到所有关系类型的事件入口 |
| 看排行榜 | 点击某类事件卡 | 进入对应排行榜，看到 Top3 领奖台和完整榜单 |
| 看自己位置 | 滑到排行榜底部 | 看到自己当前关系状态，若无关系可点击 Build 建立 |

## 5. 功能需求

### 5.1 页面结构总览

原型为单页多状态结构，通过 `show-home` / `show-news` / `show-rank` 三个状态类切换页面，维护一个视图栈 `viewStack` 支持逐级返回。

```
┌─ Home 首页 (show-home) ─────────────────┐
│  状态栏                                  │
│  For You 顶部栏              [🏆 排行榜] │
│  ┌─ Laka News Banner（核心，轮播）─────┐│
│  │ 🔥 Laka News              More ›   ││
│  │ [头像] ›› 💞 ›› [头像]              ││
│  │  · · · ·                            ││
│  └─────────────────────────────────────┘│
│  用户信息流（profile cards）             │
│  底部导航 Home/Moment/Party/Msg/Mine     │
└──────────────────────────────────────────┘
        │ 点 Banner
        ▼
┌─ Laka News 事件墙 (show-news) ──────────┐
│  ‹ 返回        Laka News                │
│  ┌─ Guardian Relationship 卡 ──────────┐│
│  │ [头像] ›› 🛡️ ›› [头像]   Go to see ›││
│  └─────────────────────────────────────┘│
│  ┌─ CP Relationship 卡 ────────────────┐│
│  │ [头像] ›› 💞 ›› [头像]   Go to see ›││
│  └─────────────────────────────────────┘│
└──────────────────────────────────────────┘
        │ 点事件卡
        ▼
┌─ 关系排行榜 (show-rank) ────────────────┐
│  ‹ 返回   Guardian/CP     ? ↻           │
│  [Daily] Weekly  Monthly                │
│  ◷ 倒计时 23:33:33                       │
│  ┌─ 前三名领奖台（2-1-3）──────────────┐│
│  │   ②        ①        ③              ││
│  │ [双头像] [双头像] [双头像]           ││
│  │  分数    分数    分数               ││
│  └─────────────────────────────────────┘│
│  4. [双头像] 名字  分数                  │
│  5. ...                                  │
│  ~ [自己] Your status → Build            │
└──────────────────────────────────────────┘
```

### 5.2 首页（Home）

#### 5.2.1 状态栏
- **展示内容**：左侧时间（如 18:52），右侧网络状态（5GA）+ 电量（数字 + 深色胶囊背景）。
- **说明**：原型为静态示意，实际应由系统状态栏接管，此处仅占位。

#### 5.2.2 For You 顶部栏
- **左侧**：页面标题「For You」，字号 28px，字重 900，颜色 `#211139`，标题下方有紫色下划线装饰（宽 118px，高 5px，圆角 99px，色 `#6237ee`）。
- **右侧**：奖杯按钮 🏆，圆形（50×50px），白底，金色图标 `#ffb72e`，带阴影。点击进入**关系排行榜**（直跳第三级，跳过事件墙）。
- **交互**：奖杯按钮是排行榜的快捷入口，与「Banner → 事件墙 → 排行榜」路径并存。

#### 5.2.3 ⭐ Laka News Banner（核心模块）

> 这是本次需求的重点新增模块，位于顶部栏与用户信息流之间。

**模块结构**：
- 卡片容器：圆角 22px，白底，阴影 `0 14px 30px rgba(21,16,42,.12)`，最小高度 150px。
- 顶部行：
  - 左：`🔥 Laka News` 热门标签（字号 16px，字重 950）。
  - 右：`More` 更多入口，带向下箭头符号（ › 形态，CSS border 旋转实现）。
- 舞台区（Stage）：
  - 三列网格布局：`98px | 1fr | 98px`。
  - 左：发起人头像 + 昵称。
  - 中：关系流转示意 `›› [关系图标] ››`，关系图标根据类型切换（CP=💞，Guardian=🛡️）。
  - 右：接收人头像 + 昵称。
  - 底部：圆点指示器，数量 = 事件总数，当前项高亮。
  - 关系类型标签：左上角显示当前事件类型（CP / Guard），随轮播切换。
  - 背景渐变：CP 类型为粉紫径向渐变 `radial-gradient(circle at 94% 50%, rgba(243,147,255,.42), ...)`，Guardian 类型为另一套配色。

**数据轮播**：
- 轮播间隔：**2500ms**（原型硬编码，可配置）。
- 轮播逻辑：按 `items` 数组顺序循环，`idx = (idx+1) % length`。
- 每次切换更新：左右头像、左右昵称、关系图标、关系类型标签、类型配色 class（`type-cp` / `type-guard`）、圆点高亮态。

**数据结构（单条事件）**：
```js
{
  from: 'Lina',          // 发起人昵称
  to: 'Sara',            // 接收人昵称
  type: 'cp',            // 关系类型：cp | guard
  label: 'CP',           // 类型展示标签
  icon: '💞',            // 关系图标
  fromAvatar: 'url',     // 发起人头像
  toAvatar: 'url'        // 接收人头像
}
```

**交互**：
- 点击 Banner 任意区域 → 进入「Laka News 事件墙」。
- 「More」入口视觉上独立，但原型中点击同样进入事件墙（后续可拆分为独立 More 落地页）。

**边界与异常**：
- 事件列表为空：Banner 隐藏，信息流自然上移（参考注释 `feed-list` 说明）。
- 仅 1 条事件：不轮播，圆点指示器隐藏或仅显示 1 个。
- 头像加载失败：需有默认占位头像（原型未处理，开发需补充）。
- 昵称过长：需省略号截断（原型未明确，建议单行省略）。

#### 5.2.4 用户信息流（Feed）
- **位置**：Banner 下方。
- **内容**：用户 profile 卡片列表，含头像、昵称（带 ✓ 认证标）、徽章（性别♀、等级△/☆）、bio、Hi 按钮，大卡可含照片画廊。
- **与 Banner 的关系**：Banner 隐藏或无数据时，信息流需自然上移填补（不预留空白）。
- **头像资源**：使用 MENA 风格女性头像（见全局规则）。

#### 5.2.5 底部导航
- 固定底部，5 个 Tab：Home（❤，当前选中）/ Moment（◒）/ Party（⌂）/ Message（▢）/ Mine（◡）。
- 切换页面时不遮挡内容（已通过 `.page` 的 `padding-bottom: 92px` 预留空间）。

### 5.3 Laka News 事件墙（show-news）

#### 5.3.1 顶部栏
- 左：返回按钮 `‹`（id: `newsBack`）。
- 中：标题「Laka News」。
- 右：占位 spacer（预留设置入口位）。

#### 5.3.2 事件卡片列表
- 按关系类型分类展示，原型含两类：
  - **Guardian Relationship 卡**（`data-rank="guard"`）：盾牌关系。
  - **CP Relationship 卡**（`data-rank="cp"`）：CP 关系。
- 卡片结构：
  - 头部行：类型标题（如「Guardian Relationship」）+ 「Go to see ›」引导。
  - 主体：左头像 + 昵称 ›› 关系图标 ›› 右头像 + 昵称。

#### 5.3.3 交互
- 点击返回 `‹`：从视图栈弹出上一页；若栈空则回首页。
- 点击事件卡：进入对应类型排行榜（`openRank(type)`）。

#### 5.3.4 边界与异常
- 事件类型扩展：未来可能新增更多关系类型（如 Friend / Bestie），卡片结构需支持动态扩展。
- 无某类事件：对应卡片隐藏，而非展示空状态（建议）。

---

### 5.4 关系排行榜（show-rank）

#### 5.4.1 顶部栏
- 左：返回按钮 `‹`（id: `rankBack`），回到事件墙。
- 中：类型标签（Guardian / CP，随入口动态切换）。
- 右：工具栏 `?`（帮助）+ `↻`（刷新）。原型中为静态图标，未绑定功能。

#### 5.4.2 周期切换
- 三个 Tab：**Daily**（默认选中）/ Weekly / Monthly。
- 原型中为静态展示，未绑定切换逻辑，开发需实现切换并刷新榜单数据。

#### 5.4.3 倒计时
- 展示 `◷ 23:33:33` 形态的倒计时，表示当前周期结束剩余时间。
- 原型为静态值，开发需接入真实倒计时逻辑（每秒更新）。

#### 5.4.4 前三名领奖台（Podium）
- 布局：**2 - 1 - 3**（第二名居左、第一名居中且最高、第三名居右）。
- 每位展示：
  - 双头像组合（`pair-avatars`，两个头像并列）。
  - 关系标签 pill（CP / Guard，带类型配色）。
  - 用户名（两行，双用户昵称）。
  - 分数 `🔥 {score}`（单位 m）。
  - 名次 medal（1 / 2 / 3）。
- 第一名视觉上最突出（高度最高、尺寸最大）。

#### 5.4.5 排行榜列表（第 4 名起）
- 行结构：名次序号 + 双头像迷你组合（`mini-pair`）+ 关系 pill + 用户名（两行）+ 分数。
- 原型展示 6 行示例数据（第 4~9 名）。

#### 5.4.6 当前用户状态行（Self State）
- 位置：列表最底部。
- 结构：序号 `~` + 自己头像 + 关系占位符 + 「Your status」标题 + 副标题「No {类型} relationship yet」+ `Build` 按钮。
- 交互：点击 `Build` 引导用户建立该类型关系（原型未绑定，开发需对接建立关系流程）。
- 边界：若用户已有该关系，则展示自己的真实排名与分数，隐藏 Build 按钮（原型未覆盖此态，需补充）。

#### 5.4.7 数据结构
```js
rankData = {
  cp: {
    label: 'CP',
    pill: 'CP',
    icon: '💞',
    title: 'CP Relationship Ranking',
    top: [  // 前三名，每项 [用户A名, 用户B名, 分数, 头像A, 头像B]
      ['Adeline Bryan', 'Julian Keller', '1321m', urlA, urlB],
      ...
    ]
  },
  guard: { ... }
}
// 第4名起列表
rowNames = [['Heather Clark','Ella Lamb','998'], ...]
```

## 6. 交互流程与页面跳转

### 6.1 跳转关系
```
[Home 首页]
   ├─ 点 Banner ──────────→ [Laka News 事件墙]
   │                            ├─ 点 Guardian 卡 ─→ [Guardian 排行榜]
   │                            ├─ 点 CP 卡 ───────→ [CP 排行榜]
   │                            └─ 点 ‹ 返回 ──────→ [Home 首页]
   └─ 点 🏆 奖杯按钮 ─────→ [排行榜]（直跳，跳过事件墙）
                            
[排行榜]
   └─ 点 ‹ 返回 ──────────→ [Laka News 事件墙]
```

### 6.2 视图栈机制
- 维护 `viewStack`，初始 `['show-home']`。
- 进入新页：当前页压栈，切换到目标页。
- 返回：弹栈回到上一页；栈空则回首页（fallback `show-home`）。
- 奖杯按钮直跳排行榜：当前页压栈后切到 `show-rank`，返回时回首页。

### 6.3 动效
- Banner 轮播：每 2.5s 自动切换，无手动滑动（原型未实现手动滑，建议后续补充左右滑手势）。
- 页面切换：原型未做过渡动效，建议补充 push/pop 转场（开发可选）。

## 7. 数据需求

### 7.1 接口清单（建议，需后端确认）
| 接口 | 用途 | 入参 | 出参 |
|---|---|---|---|
| 关系事件列表 | Banner 轮播 + 事件墙 | 无 / 分页 | `items[]`（见 5.2.3 结构） |

### 7.2 本地存储
- 原型注释系统使用 `localStorage` key：`prototype-annotation-editor-draft:{prototypeId}`，仅用于原型交付工具，**不属于产品功能**。

### 8.2 关系类型配色
| 类型 | 图标 | 配色特征 |
|---|---|---|
| CP | 💞 | 粉紫径向渐变 `rgba(243,147,255,.42)` |
| Guardian | 🛡️ | 另一套配色（原型 CSS 中 `type-guard` 定义，开发从样式表提取） |

### 8.3 字体与字号
- 字体族：`Inter, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`。
- 字号梯度：12 / 14 / 15 / 16 / 20 / 25 / 28px。
- 字重：800 / 850 / 900 / 950（重度使用粗体，符合 MENA 视觉偏好）。

### 8.4 尺寸
- 设备：393 × 852px（iPhone 16）。
- 头像：98×98px（舞台区）、小尺寸（事件卡、排行榜）。
- 奖杯按钮：50×50px。
- 底部导航高度预留：92px。

### 8.5 可访问性（WCAG AA）
- 文字与背景对比度 ≥ 4.5:1（`#211139` on `#fff` / `#f4f3f6` 满足）。
- 可点击元素最小触控区 ≥ 44×44px（奖杯按钮 50px 满足；Banner 整体可点满足）。
- 返回按钮、图标按钮需补充 `aria-label`（原型未完整，开发需补）。

## 9. 边界与异常

| 场景 | 处理方式 |
|---|---|
| Banner 无事件数据 | 隐藏 Banner，信息流上移 |
| Banner 仅 1 条事件 | 不轮播，圆点隐藏或单点 |
| 头像加载失败 | 默认占位头像（开发补充） |
| 昵称过长 | 单行省略号 |
| 事件墙某类无事件 | 隐藏对应卡片 |
| 排行榜无数据 | 空状态提示（原型未覆盖，需补充） |
| 用户已有关系（排行榜底） | 展示真实排名，隐藏 Build 按钮（原型未覆盖，需补充） |
| 网络异常 | 各页加载失败兜底（原型未覆盖，需补充） |
| 倒计时结束 | 切换到下一周期并刷新榜单 |

### 10.1 原型注释工具说明
原型内置一个「原型注释」编辑面板（`annotation-panel`），**这是设计交付辅助工具，不属于产品功能**，用途：
- 高亮原型中的 14 个模块，逐个编辑交付说明（title + note）。
- 支持保存到服务端 `/api/annotations`、恢复默认、导出带注释的 HTML。
- 导出文件命名：`homepage-news-wall-annotated-{activeKey}-{timestamp}.html`。

> 正式开发实现时，注释面板相关代码（`annotation-panel`、`annotation-export-store`、底部 script）应剥离，不进入生产环境。
