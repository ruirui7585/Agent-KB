## 项目理解（首个会话：通读全部核心文件）
- 项目：通用版高保真 HTML 原型讲解器（universal-version-html），本地 MVP 工具。
- 技术栈：纯前端原生 ES Module（无框架）+ Node 轻量 HTTP 服务（server.js，port 5173）。
- 核心文件职责：
  - `server.js`：静态服务 + `/api/prototype-projects` 上传导入（formidable 解析、adm-zip 解压、路径穿越防护、静态原型校验）。
  - `app.js`：主控制器（~900 行单文件），内存状态 + localStorage 草稿。
  - `storage.js`：localStorage 持久化与 project 归一化。
  - `exporter.js`：导出 JSON + 生成单文件分享 HTML（数据内联、Blob 渲染原型）。
  - `prototype-bridge.js`：上传/加载原型、读取 PROTOTYPE_CONFIG、applySceneToPrototype。
  - `ai-helper.js`：本地规则模拟的 AI 润色/补全/冲突检测（未接外部 API）。
  - `demo-data.js`：示例 IM 聊天原型 + 示例项目。
- 数据模型：functions / sceneGroups / scenes / annotations / settings，三层 功能→分组→场景，标注挂场景（百分比坐标）。
- 四种场景进入方式：state（setPrototypeState）、url、guide、note。

## 修复：完整原型文件夹导入（本次任务）
- 现象：「导入文件夹」打开的是单文件选择器，只导入 index.html，CSS/JS/图片缺失。
- 根因：`folderFileInput` 虽带 `webkitdirectory` HTML 属性，但部分浏览器仅认 JS 属性方式，导致目录选择未生效、回退成普通文件选择。
- 改动（仅限文件夹导入，未动其他逻辑）：
  1. `index.html`：清理 `folderFileInput` 无效属性（mozdirectory/msdirectory/odirectory），保留 `webkitdirectory directory multiple`。
  2. `app.js` bindEvents：显式 `els.folderFileInput.webkitdirectory = true; directory = true; multiple = true;` 强化目录模式。
  3. `app.js` handleFolderImport：若文件无 `webkitRelativePath`（浏览器不支持目录选择/用户选了单文件），提示"请选整个文件夹，或改用导入 ZIP"，避免残缺导入。
- 验证：Playwright 真实 Chromium 目录选择 → fileCount=6，style.css/app.js/png/子目录全部 200，iframe 内 app.js 执行成功，高保真页面完整还原。
- 注：`htmlFiles` 字段按设计只列 HTML；非 HTML 资源仍完整落盘并加载（serverside stripCommonRoot 保留子目录结构已确认）。

## 修复：页面无法点击（标注模式下原型交互不可用）

- 现象：导入原型后，默认"标注模式"下，点击原型按钮/链接无响应。
- 根因：标注模式 `.annotate-mode` 将 iframe 设为 `pointer-events: none`，标注层 `pointer-events: auto` 且 `inset:0` 全覆盖画布，所有点击被标注层吞掉、原型无法交互。
- 修复思路：默认不锁定原型（标注层 `pe:none`、iframe `pe:auto`），标注改为由显式操作（按钮 + 键盘 Alt/Shift）触发。
- 具体改动：
  1. **styles.css**：`annotate-mode iframe` 从 `pe:none` → `pe:auto`；新增 `.annotate-armed iframe { pe:none }` / `.annotate-armed .annotation-layer { pe:auto }`。新增 `.annotate-hint` 样式。
  2. **index.html**：工具栏添加 `#annotateToolBtn`（标注工具按钮）和 `#annotateHint` 提示。
  3. **app.js**：`els` 新增 `annotateToolBtn`/`annotateHint`；新增 `armAnnotation(on)` 切换 `annotate-armed` class + 按钮 active 状态 + 提示显隐；新增 `onKeyAnnotate` 支持 Alt/Shift 武装；`onAnnotationLayerClick` 加 `annotate-armed` 守卫；`setMode` 末尾调用 `armAnnotation(false)` 自动解除。
- 验证（Playwright 真实 Chromium，7 项测试全部 PASS）：
  - 默认标注模式 → 画布顶层为 IFRAME（交互正常）
  - 未开标注工具，点击画布 → 不创建标注
  - 标注工具按钮 → `annotate-armed`、按钮 active、提示显示
  - 标注工具开启，点击画布 → 标注创建成功（1→2）
  - 关闭标注工具 → 点击不创建标注
  - 体验模式 → iframe 交互、标注层不抓
  - Alt 按下/释放 → 武装/解除正确
