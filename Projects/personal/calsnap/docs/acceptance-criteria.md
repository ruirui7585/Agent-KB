## E2E-4：错误处理与边界

| # | 场景 | 操作 | 预期效果 |
|---|---|---|---|
| E2E-4.1 | 无网络拍照 | 断网 → 拍照 → 确认 | 显示"识别失败"明确提示 + "重试"按钮，不是白屏或 spinner 卡死 |
| E2E-4.2 | API Key 未配 | 服务器不配置 Key → 拍照 | 提示服务器未配置 `DASHSCOPE_API_KEY` |
| E2E-4.3 | 百练返回异常 | （模拟后端返回非 JSON）| 显示"识别失败"，不是 JS 报错白屏 |
| E2E-4.4 | 拍非食物图 | 拍一张风景/人脸 | 百练返回空食物数组 → 前端提示换一张图片，不保存记录 |
| E2E-4.8 | 无访问码访问 | 清空浏览器本地数据后打开公网地址 | 首页直接加载，不出现访问码输入页 |
| E2E-4.5 | 空日详情 | 看一个没有记录的日子 | 四餐均显示"无记录"或"还没记"，日总计为 0 |
| E2E-4.6 | 设置页必填 | 身高留空点保存 | 前端阻止并提示，不发送请求 |
| E2E-4.7 | 输入非法值 | 体重填 -1 或 abc | 前端阻止，"请输入有效数字" |
| E2E-4.9 | 克重输入无效 | 食物名为空、克重为 0 或超过 5000g | 前端阻止请求并指出具体项目；服务端同样返回 400 |

### E2E-4 通过标准
> 所有异常情况有明确中文提示，不白屏、不死循环、不静默失败。

## DATA-1：数据准确性

| # | 检查项 | 验证方法 | 预期值 |
|---|---|---|---|
| DATA-1.1 | BMR 男 | 输入男/175cm/70kg/28岁 | BMR = 10×70 + 6.25×175 - 5×28 + 5 = 700 + 1093.75 - 140 + 5 = **1658.75** |
| DATA-1.2 | TDEE 轻度活动 | BMR × 1.375 | 1658.75 × 1.375 = **2280.78** |
| DATA-1.3 | BMR 女 | 输入女/160cm/55kg/25岁 | BMR = 10×55 + 6.25×160 - 5×25 - 161 = 550 + 1000 - 125 - 161 = **1264** |
| DATA-1.4 | 热量差 | 今日摄入 1500，TDEE 2280 | 热量差 = 1500 - 2280 = **-780**（显示绿色） |
| DATA-1.5 | 日总计 | 早餐 480 + 午餐 620 + 晚餐 700 + 加餐 200 | 首页显示 **2000** |
| DATA-1.6 | 日均统计 | 7 天数据：2000,1800,2200,1900,2100,1700,2300 | 日均 = (2000+1800+2200+1900+2100+1700+2300)/7 = **2000** |
| DATA-1.7 | 保留精度 | 所有热量值显示 | 整数，无小数（kcal 不需要小数精度） |

### DATA-1 通过标准
> 8 组手动计算结果与 App 显示值完全一致，误差 = 0。

## 验收流程

```
1. 代码写完 + 部署到服务器
2. 逐条执行 E2E-1 → E2E-4（在 iPhone 上实际操作）
3. 人工检查 VIS-1（截图对比 UI 文档）
4. 手动计算核对 DATA-1（纸笔算 8 组数据 vs App 显示）
5. 秒表测 PERF-1
6. 交叉检查 CONSIST-1（拍照→首页→历史三处数据对比）
7. 逐条验证 DEPLOY-1
8. 出验收报告：通过/有条件通过/不通过 + 未达标项清单
```
