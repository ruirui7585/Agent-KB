# CalSnap PRD

## 一、产品概述

| 项目 | 内容 |
|---|---|
| 产品名称 | CalSnap |
| 一句话描述 | 拍照估算或按实际克重计算卡路里的饮食记录工具 |
| 目标用户 | 个人使用（后续可扩展家庭成员） |
| 平台 | PWA（Safari 添加到桌面，iOS/Android 均可） |

## 三、用户故事

### 3.1 拍照记录

```
用户在吃饭前 → 打开 CalSnap → 点拍照 → 对准食物按快门
→ 界面显示「正在识别...」→ 返回后展示结果：
  - 番茄炒蛋 180 kcal
  - 米饭 200 kcal
  - 总计 380 kcal
→ 核对/修改食物和份量 → 选择餐别（午餐）→ 确认保存 → 跳到首页看今日汇总
```

### 3.2 查看今日

```
用户打开首页 → 看到：
  - 绿色圆环进度条：今日摄入 1100 / 目标 2100 kcal
  - 还剩 1000 kcal
  - 早餐 480 kcal ✔
  - 午餐 620 kcal ✔
  - 晚餐 未记录
  - 加餐 未记录
  - 底部 7 天趋势柱状图
```

### 3.3 查看历史

```
用户点历史 Tab → 看到 30 天每日汇总列表
→ 点某天 → 展开当天四餐详情
→ 切换到统计视图 → 周/月趋势折线图 + 蛋白质/脂肪/碳水占比
```

### 3.4 设置个人资料

```
用户首次使用 → 点设置 Tab → 填写：
  - 性别（男/女）
  - 身高（cm）
  - 体重（kg）
  - 年龄
  - 活动量（久坐/轻度/中度/重度/运动员）
→ 系统自动计算并展示：每日热量消耗 ≈ 2100 kcal，建议摄入 2100 kcal
→ 用户也可以手动改目标
```

## 四、热量计算公式

### BMR（基础代谢率）- Mifflin-St Jeor

| 性别 | 公式 |
|---|---|
| 男 | 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 + 5 |
| 女 | 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄 - 161 |

### TDEE（每日总消耗）= BMR × 活动系数

| 活动量 | 系数 |
|---|---|
| 久坐（几乎不运动） | 1.2 |
| 轻度（每周 1-3 天） | 1.375 |
| 中度（每周 3-5 天） | 1.55 |
| 重度（每周 6-7 天） | 1.725 |
| 运动员（每天高强度） | 1.9 |

### 热量差

```
热量差 = 今日摄入 - TDEE
正数 = 热量盈余（会增重）
负数 = 热量缺口（会减重）
默认目标 = TDEE（维持体重）
用户可手动调整目标值
```

## 五、数据模型

```
UserProfile（单条，设置页管理）
├── gender: 'male' | 'female'
├── height_cm: number
├── weight_kg: number
├── age: number
├── activity_level: 'sedentary' | 'light' | 'moderate' | 'active' | 'very_active'
├── tdee: number（自动计算）
└── daily_cal_goal: number（默认 = tdee，用户可改）

FoodRecord
├── id: UUID
├── date: '2026-06-15'
├── time: '12:30'
├── meal_type: 'breakfast' | 'lunch' | 'dinner' | 'snack'
├── foods: [{name, calories, weight_g}]
├── total_cal: number
├── image_path: string（可选，保存图片）
├── source: 'vision'（v2 加 'barcode' | 'manual'）
└── created_at: datetime
```

## 六、页面结构（5 个 Tab）

```
TabBar
├── 首页    首页 · 今日汇总 + 热量圆环 + 四餐概览 + 周趋势图
├── 拍照    拍照 · 系统相机/相册 + 餐别选择 + 识别结果展示
├── 历史    历史 · 30 天列表 → 天详情 → 统计图表
├── 扫码    扫码 · (MVP 占位，v2 启用)
└── 设置    设置 · 个人资料 + 热量目标 + 百练状态 + 服务器状态
```

## 七、非功能需求

| 项目 | 要求 |
|---|---|
| 响应速度 | 拍照后 5 秒内出结果 |
| 离线 | PWA Service Worker，无网络时展示缓存壳 |
| 安全 | 百练 Key 仅存在服务器环境变量；公网识别接口启用 IP 限流 |
| 跨设备 | 单用户本地使用，暂不需要 |
| 隐私 | 图片仅转发百练完成识别，不写入本地数据库或长期存储 |

## 八、中餐识别优化策略

- 百练识别 prompt 要求输出中文菜名、份量、热量区间和置信度
- 混合菜品（如盖浇饭、拌面）提示 AI 分别列出主料
- 常见中餐热量参考内置到 prompt 上下文
- v2 考虑对接中国食物成分表数据库
