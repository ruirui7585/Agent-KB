# Changelog

## 2026-07-01 - 趋势页多指标图表

### Added

- 趋势页增加热量、脂肪变化、体脂率、体重四个指标 Tab。
- 增加周、月、3 个月、年度四个时间范围。
- 脂肪变化使用每日热量差折算；体脂率与体重曲线以当前资料为锚点，结合历史热量差反推。
- 服务端统计范围扩展到 365 天，并增加输入范围校验。

### Risks

- 体脂率和体重曲线是模型估算，不是历史实测；水分、肌肉与代谢适应不会被模型捕获。
- 未完整记录饮食或运动的日期按无变化处理，可能低估真实波动。

## 2026-07-01 - 首页脂肪与体脂趋势预估

### Added

- 首页将当日热量差按 `7700 kcal ≈ 1 kg 脂肪`折算为今日和 7 天脂肪能量等价。
- 基于 BMI、年龄和性别生成当前体脂率估算，并展示 7 天、30 天线性趋势。
- 资料不完整时展示完善资料引导，不生成虚假体脂结果。
- 体脂与脂肪换算纯函数测试。

### Risks

- 预测假设热量差保持不变，且体重变化主要来自脂肪；实际结果会受水分、肌肉、代谢适应和记录误差影响。
- 该结果仅用于趋势参考，不替代体脂秤或医学测量。

## 2026-07-01 - 双模式饮食记录

### Added

- “拍照估算 / 按克重计算”二选一入口。
- 克重模式支持一餐添加 1–10 种食物，并按实际重量计算热量和三大营养素。
- `POST /api/food/calculate`，计算成功后仍需用户确认才保存。

### Changed

- 拍照页升级为“记录饮食”，优化模式选择、餐别选择、输入表单和结果来源标识。
- 图片与克重模式共用分阶段等待、取消、重试和结果校对流程。

### Risks

- 即使克重准确，品牌、可食部比例与烹饪用油仍会造成热量偏差。

## 2026-07-01 - 移除访问码

### Changed

- 移除服务端 `APP_ACCESS_TOKEN` 校验和前端 `X-App-Token` 请求头。
- 移除首次访问输入页及“我的”页访问码配置。
- 公网接口改为直接访问，食物识别继续保留 IP 频率限制。

### Risks

- 知道服务器公网 IP 的访问者可以读取或修改当前个人数据，也可以发起受限频率内的百练识别请求。

## 2026-06-30 - 高保真能量平衡版

### Added

- 运动记录：类型、时长、强度、MET 自动估算和手动修正。
- 每日基础消耗、运动消耗、总消耗与热量差字段。
- 首页、记录、趋势、运动页和中央快捷添加面板。
- 运动数据表、接口、统计聚合与自动测试。

### Changed

- 移动端视觉升级为 430px 高保真卡片布局和固定底部导航。
- 热量差统一为 `摄入 - 基础日常消耗 - 运动消耗`。
- 基础日常消耗固定使用久坐系数，避免与单独运动重复计算。

### Risks

- 运动消耗为 MET 估算，不替代专业穿戴设备数据。
- 未填写个人资料时，基础消耗默认采用 1400 kcal。

## 2026-06-30 - CalSnap 百练主线收口

### Added

- 百练 `qwen3.7-plus` 主模型与 `qwen3.6-flash` 回退模型。
- 食物热量区间、营养素和置信度字段。
- 用户修正食物名称、份量、热量和删除误识别项。
- 确认后保存接口、可选访问码、识别限流和纯函数测试。

### Changed

- 活动实现统一为 `web/ + server/`。
- 产品品牌由 FoodMind 统一为 CalSnap。
- API Key 改为仅从服务器环境变量读取。
- PWA API 请求不再进入 Service Worker 缓存。

### Fixed

- 识别上传即写数据库的问题。
- 原始图片落盘与隐私口径不一致的问题。
- TDEE 示例数值不一致。
- 百练文本进入 HTML 属性时转义不完整的问题。
- 使用 Node.js 内置 `randomUUID()` 替代存在安全公告的旧 UUID 依赖。
- 增加 PM2 CommonJS 配置，兼容 `type: module` 项目的生产启动。
- 重新编码 PWA 图标，修复部分 PNG 解码器无法读取的问题。

### Deployed

- 已部署至阿里云轻量应用服务器 `47.254.214.153`。
- 百练真实图片调用、访问码保护、PM2 自启、Nginx 与公网浏览器验收通过。
- 当前为 HTTP 临时入口；绑定域名后再启用 HTTPS 与正式 PWA 安装。

### Risks

- 单张照片无法精确判断重量、用油量和隐藏配料，必须保留用户确认。
- 云端百练真实调用与 HTTPS 部署仍需用户提供新 Key、业务空间 ID、实例和域名。
