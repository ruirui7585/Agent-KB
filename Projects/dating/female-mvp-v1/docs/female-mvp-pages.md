# Female Operator MVP Pages

## Product Positioning

Female Operator App is a staff workspace, not a normal female user app. Staff can operate multiple backend-configured model profiles, reply to aggregated male user conversations, and drive conversion through scripts, photos, private content, and call invites.

## Page Tree

- Login
- Inbox
- Chat Detail
- Models
- Model Detail
- Content
- Content Picker / Send Preview
- Me
- Work Status

## Core Fields

- Login: staff account, password, demo login.
- Inbox: staff state, daily metrics, conversation cards with male user and model identity.
- Chat Detail: male payment status, free messages left, model reply identity, suggested action, messages, tools.
- Models: bound model count, online model count, model cards.
- Model Detail: profile preview, persona notes, content pack, today data, read-only notice.
- Content: filters, content owner, type, price, sent status, usage scenario.
- Content Picker: preview, current conversation, duplicate warning, send actions.
- Me: staff profile, today summary, work info.
- Work Status: online/offline status and switch controls.

## Core Click Paths

- Login to Inbox.
- Bottom tabs to Inbox, Models, Content, Me.
- Inbox card to Chat Detail and back.
- Models card to Model Detail and back.
- Content card to Content Picker and back.
- Chat Detail Private to Content Picker, then send to Chat Detail.
- Me to Work Status and back.

## MVP Exclusions

- No male app pages.
- No full task system.
- No full earnings settlement.
- No QA backend.
- No complex permissions.
- No full scheduling.
- No AI auto-reply.
- No handoff system.
- No model profile editing.
- No content upload or audit.
